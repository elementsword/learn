深与浅拷贝

**浅拷贝（shallow copy）与深拷贝（deep copy）**
 是 C++ 对象管理中最常见、也最容易出 bug 的地方之一。
 尤其是你后面会写含有指针、动态内存的类（比如图像帧、音频缓存、数据块等），
 如果不理解这一块，内存错误非常容易出现。

------

## 🧩 一、拷贝是什么？

当你把一个对象“赋值给另一个对象”时，比如：

```
A a1;
A a2 = a1;  // 或 A a2(a1);
```

编译器就会去 **拷贝对象的成员变量**。

如果你没写自己的拷贝逻辑，编译器会**自动生成一个“默认拷贝构造函数”**，
 这个默认行为就是 **浅拷贝**。

------

## 🧠 二、浅拷贝（Shallow Copy）

👉 “按字节拷贝成员变量的值”
 包括指针在内，都是直接复制指针值。

```
class A {
public:
    int* data;
    A() { data = new int(10); }
    ~A() { delete data; }
};

int main() {
    A a1;
    A a2 = a1;  // 默认浅拷贝（编译器生成）
}
```

问题：

- `a1.data` 和 `a2.data` 指向 **同一块堆内存**
- 当 `a1` 和 `a2` 析构时，都会 `delete data`
- 🔥 **二次释放（double free）错误！**

------

## ✅ 三、深拷贝（Deep Copy）

👉 “复制数据本身，而不是指针地址”

你要手动写拷贝构造函数，让每个对象有自己的独立资源。

```
class A {
public:
    int* data;
    A() { data = new int(10); }
    ~A() { delete data; }

    // ✅ 深拷贝构造
    A(const A& other) {
        data = new int(*other.data);
    }
};
```

这样：

```
A a1;
A a2 = a1;  // 深拷贝
*a2.data = 20;

cout << *a1.data;  // 仍是 10，不影响 a1
```

------

## ⚙️ 四、赋值运算符的深拷贝

除了构造时的拷贝，还要考虑 **对象已存在时被赋值**：

```
class A {
public:
    int* data;
    A() { data = new int(10); }
    ~A() { delete data; }

    // 深拷贝赋值运算符
    A& operator=(const A& other) {
        if (this == &other) return *this; // 自赋值检测
        delete data;                      // 释放旧资源
        data = new int(*other.data);      // 深拷贝
        return *this;
    }
};
```
