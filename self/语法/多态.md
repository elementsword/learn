## 🧩 一、什么是多态（Polymorphism）

> “同一接口，不同表现。”

简单来说：
 👉 通过 **基类指针（或引用）** 去调用 **派生类的函数**，
 而实际执行的行为，**取决于对象的真实类型**。

------

### 🔸 例子：没有多态时

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    void speak() { cout << "Animal 叫\n"; }
};

class Dog : public Animal {
public:
    void speak() { cout << "狗汪汪\n"; }
};

int main() {
    Animal* p = new Dog();
    p->speak();  // ❌ 输出 "Animal 叫"，而不是 "狗汪汪"
}
```

👉 因为普通成员函数是**静态绑定（static binding）**，
 编译时就决定了调用哪个函数。

------

### ✅ 使用多态（虚函数）

```
class Animal {
public:
    virtual void speak() { cout << "Animal 叫\n"; }
    virtual ~Animal() {} // ✅ 基类析构函数也要设为虚的
};

class Dog : public Animal {
public:
    void speak() override { cout << "狗汪汪\n"; }
};

int main() {
    Animal* p = new Dog();
    p->speak(); // ✅ 输出 "狗汪汪"
    delete p;   // ✅ 调用 Dog 析构函数
}
```

👉 因为 `speak()` 是虚函数，
 调用时会进行 **动态绑定（dynamic dispatch）**，
 运行时根据实际对象类型选择函数。

------

## 🧠 二、实现机制（vtable）

C++ 实现多态的底层机制是：

| 组件                     | 含义                                                         |
| ------------------------ | ------------------------------------------------------------ |
| **vptr**（虚函数表指针） | 每个含虚函数的对象里都有一个隐藏指针，指向类的虚函数表。     |
| **vtable**（虚函数表）   | 编译器为每个有虚函数的类生成的函数指针表。表项指向当前类的虚函数实现。 |

🔹 调用过程：

```
p->speak()
↓
p（对象）中保存了 vptr
↓
vptr 指向类的虚表
↓
查表找到 speak 对应的函数地址
↓
调用 Dog::speak()
```

------

### 💡 虚函数表示意图：

```
Animal 对象：
[ Animal::vptr ] → [ Animal::speak(), ~Animal() ]

Dog 对象：
[ Dog::vptr ] → [ Dog::speak(), ~Dog() ]
```

所以即使你拿 `Animal*` 指向 `Dog`，
 vptr 依然指向 `Dog` 的虚表，从而实现“多态”。

------

## ⚙️ 三、构造与析构中的多态

- **构造函数中调用虚函数** → 不会多态！

  ```
  class A {
  public:
      A() { speak(); }
      virtual void speak() { cout << "A speak\n"; }
  };
  
  class B : public A {
  public:
      void speak() override { cout << "B speak\n"; }
  };
  
  int main() { B b; }  // 输出：A speak
  ```

  👉 因为此时对象还没完全构造为 `B`，vptr 仍指向 `A` 的表。

------

## 🧾 四、析构函数与多态

一定记住：

> 🔥 **基类析构函数要声明为 `virtual`！**

否则通过基类指针 `delete` 派生类对象会出现 **内存泄漏**。

```
class Base {
public:
    virtual ~Base() { cout << "Base 析构\n"; }
};
class Derived : public Base {
public:
    ~Derived() { cout << "Derived 析构\n"; }
};

int main() {
    Base* p = new Derived();
    delete p;
}
// 输出：
// Derived 析构
// Base 析构
```

## 🧮 五、多态的三要素

| 要素          | 含义                       |
| ------------- | -------------------------- |
| 继承          | 必须有继承关系             |
| 虚函数        | 必须用 `virtual` 修饰      |
| 基类指针/引用 | 通过基类类型访问派生类对象 |

# 🧩 C++ 静态多态（Static Polymorphism）全总结

静态多态发生在 **编译期**，编译器根据类型或函数签名直接决定调用哪个函数，**不依赖运行时虚表**。
 主要包括以下三种形式 👇

------

## 🧠 1️⃣ 函数重载（Function Overloading）

在同一作用域中定义多个同名函数，只要**参数类型或个数不同**。

### ✅ 示例：

```
#include <iostream>
using namespace std;

void print(int x) {
    cout << "打印整数: " << x << endl;
}

void print(double x) {
    cout << "打印浮点数: " << x << endl;
}

void print(const string& s) {
    cout << "打印字符串: " << s << endl;
}

int main() {
    print(10);          // 调用 print(int)
    print(3.14);        // 调用 print(double)
    print("hello");     // 调用 print(const string&)
}
```

🧩 特点：

- 同名函数根据参数不同“静态决定”调用哪个。
- 编译阶段完成解析，无虚表、无运行时开销。

------

## 🧮 2️⃣ 运算符重载（Operator Overloading）

本质上是 **函数重载的一种特例**，但形式更直观。
 允许为自定义类型定义 +、-、==、[]、() 等运算符的行为。

### ✅ 示例：

```
#include <iostream>
using namespace std;

class Point {
public:
    int x, y;
    Point(int a, int b) : x(a), y(b) {}

    // 运算符重载
    Point operator+(const Point& other) const {
        return Point(x + other.x, y + other.y);
    }

    void show() const {
        cout << "(" << x << ", " << y << ")" << endl;
    }
};

int main() {
    Point a(1, 2);
    Point b(3, 4);
    Point c = a + b;   // 编译期静态绑定到 operator+()
    c.show();          // 输出 (4, 6)
}
```

🧩 特点：

- 本质是函数调用（如 `a + b` → `a.operator+(b)`）。
- 调用在编译阶段解析，不涉及虚表。
- 常用于自定义类型行为一致性（如向量、矩阵、字符串类等）。

------

## ⚙️ 3️⃣ 模板（Templates）

模板是 **泛型编程（Generic Programming）** 的核心，
 编译器会在**编译期**根据不同类型生成不同版本的函数或类。

### ✅ 函数模板：

```
#include <iostream>
using namespace std;

template<typename T>
T add(T a, T b) {
    return a + b;
}

int main() {
    cout << add(1, 2) << endl;       // 编译期生成 add<int>
    cout << add(1.1, 2.2) << endl;   // 编译期生成 add<double>
}
```

### ✅ 类模板：

```
template<typename T>
class Box {
public:
    T value;
    Box(T v) : value(v) {}
    void show() { cout << value << endl; }
};

int main() {
    Box<int> b1(10);
    Box<string> b2("hello");
    b1.show();   // 生成 Box<int>
    b2.show();   // 生成 Box<string>
}
```

🧩 特点：

- 模板实例化发生在编译期。
- 每种类型都会生成独立的代码。
- 性能高、无虚函数开销。

------

## 💡 4️⃣ （进阶）CRTP（Curiously Recurring Template Pattern）

一种高级的“编译期多态”技巧：
 通过模板参数引用派生类，实现**类层面的静态多态**。

```
#include <iostream>
using namespace std;

template<typename Derived>
class Base {
public:
    void interface() {
        static_cast<Derived*>(this)->implementation();
    }
};

class DerivedA : public Base<DerivedA> {
public:
    void implementation() {
        cout << "DerivedA 实现\n";
    }
};

class DerivedB : public Base<DerivedB> {
public:
    void implementation() {
        cout << "DerivedB 实现\n";
    }
};

int main() {
    DerivedA a;
    DerivedB b;
    a.interface();  // 调用 DerivedA::implementation()
    b.interface();  // 调用 DerivedB::implementation()
}
```

🧩 特点：

- 不用虚函数即可实现类似多态的行为。
- 完全在编译期决定，性能极高。
- 常用于数学库、物理引擎、机器学习框架中（如 Eigen、TensorFlow）。

------

## 📊 5️⃣ 对比总结表

| 类型             | 示例                            | 绑定时间 | 是否用虚表 | 性能 | 用途           |
| ---------------- | ------------------------------- | -------- | ---------- | ---- | -------------- |
| 函数重载         | `print(int)` vs `print(double)` | 编译期   | 否         | ✅ 快 | 接口灵活       |
| 运算符重载       | `a + b`                         | 编译期   | 否         | ✅ 快 | 自定义类型行为 |
| 模板             | `template<typename T>`          | 编译期   | 否         | ✅ 快 | 泛型编程       |
| CRTP             | `Base<Derived>`                 | 编译期   | 否         | ✅ 快 | 高性能类层继承 |
| 动态多态（对比） | `virtual void f()`              | 运行期   | ✅ 是       | 稍慢 | 运行时灵活性   |
