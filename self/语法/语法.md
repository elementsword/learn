## `noexcept` 是什么？

`noexcept` 是 C++11 引入的**异常说明符**（exception specifier），
 用来告诉编译器 **某个函数不会抛出异常**。

## 🧱 一、`this` 指针是什么？

当你调用一个类的成员函数时，
 编译器**在背后偷偷地给这个函数多传了一个隐藏参数** ——
 那就是 `this` 指针。

```
class Point {
public:
    int x, y;
    void show() {
        cout << "x=" << x << ", y=" << y << endl;
    }
};
```

当你写：

```
Point p{3, 5};
p.show();
```

编译器其实会偷偷变成这样：

```
show(&p);  // this = &p
```

所以在成员函数里：

- `this` → 指向调用该函数的**对象本身**
- `*this` → 就是当前对象（解引用）

------

## 🧩 二、`this` 的几个关键性质

| 特性                                             | 说明                                        |
| ------------------------------------------------ | ------------------------------------------- |
| 1️⃣ 只在**非静态成员函数**中存在                   | 静态成员函数没有对象上下文，所以没有 `this` |
| 2️⃣ 在类中隐式可用                                 | 你可以写 `x` 或 `this->x`，等价             |
| 3️⃣ `this` 是一个指针（类型为 `ClassName* const`） | 指针本身不可改，但可以修改它指向的对象成员  |

------

## 📘 三、举例说明

### ✅ （1）区分成员和形参同名时：

```
class A {
    int x;
public:
    void setX(int x) {
        this->x = x;  // 明确是成员x = 参数x
    }
};
```

如果不写 `this->`，那 `x = x;` 会赋值给自己，啥都没变。

------

### ✅ （2）链式调用（返回 *this）

```
class Point {
    int x, y;
public:
    Point(int a=0, int b=0): x(a), y(b) {}
    Point& moveX(int dx) { x += dx; return *this; }
    Point& moveY(int dy) { y += dy; return *this; }
    void show() const { cout << "(" << x << "," << y << ")" << endl; }
};

int main() {
    Point p(1,2);
    p.moveX(3).moveY(4).show();  // 链式调用
}
```

👉 因为 `moveX()` 返回 `*this`（即当前对象本身），
 才能连续 `.moveY().show()` 这样写。

------

### ✅ （3）`const` 成员函数的 `this` 类型

```
void func() const;
```

编译器会自动把 `this` 视为
 👉 `const ClassName* const this`

表示：

- `this` 本身不能变
- 也不能通过 `this` 改变成员变量

所以在 `const` 函数里不能写 `this->x = 1;`

------

## ⚙️ 四、`this` 在运算符重载中的使用

比如：

```
Point operator+(const Point& other) const {
    return Point(this->x + other.x, this->y + other.y);
}
```

这里 `this->x` 表示当前对象的 `x`，
 `other.x` 是参数对象的 `x`。

所以 `this` 常常用来区分“自己”和“别人”。
