# 一、什么是前向声明（Forward Declaration）

**前向声明**是告诉编译器：

> “有这么一个类型存在，但我现在不告诉你它内部怎么实现。”

它的核心目的是**让编译器知道类型名字**，而不必立即包含完整定义。

```
class B;  // 前向声明
```

意思：

- `B` 是一个类
- 编译器现在知道类型名字，但不知道大小、成员或方法
- 可以用在需要类型名字但不需要内部信息的地方

------

# 二、前向声明的基本用途

前向声明主要用于：

1. **减少头文件依赖**
   - 避免 `#include` 过多
   - 提升编译速度
2. **打破循环依赖**
   - 类 A 和类 B 互相引用
3. **声明接口 / API**
   - 用户只需要知道类型存在，不需要实现

# 三、前向声明能做什么（可用性规则）

## 3.1 可以做的

| 使用方式   | 示例                                         | 说明                                          |
| ---------- | -------------------------------------------- | --------------------------------------------- |
| 指针       | `B* ptr;`                                    | 编译器只需要知道指针大小，不需要知道 B 的大小 |
| 引用       | `B& ref;`                                    | 同上                                          |
| 函数参数   | `void f(B* p);`                              | 编译器只关心指针/引用类型                     |
| 函数返回值 | `B* g();`                                    | 同上                                          |
| 模板参数   | `template<typename T> class Foo; Foo<B>* p;` | 可以用于模板的指针/引用                       |

✅ **本质：只要编译器不需要知道类型的大小或内部结构，就可以使用前向声明**
