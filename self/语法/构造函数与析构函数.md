## C++ 中的构造函数种类

### 1. **默认构造函数**

- 作用：不带参数，用来初始化对象。
- 如果你没写，编译器会自动生成一个。

```cpp
class A {
public:
    A() { std::cout << "默认构造函数\n"; }
};
​
A a;   // 调用默认构造
```

### 2. **有参构造函数**

- 作用：允许对象在创建时初始化成员。

```c++
class B {
public:
    int x;
    B(int val) : x(val) {
        std::cout << "有参构造函数: x=" << x << "\n";
    }
};
​
B b(10);   // 调用有参构造
```

### 3. **拷贝构造函数**

- 作用：用一个已存在的对象，去创建一个新的对象。
- 如果你不写，编译器会自动生成“浅拷贝”。

```cpp
class C {
public:
    int x;
    C(int v) : x(v) {}
    C(const C& other) : x(other.x) {   // 拷贝构造
        std::cout << "拷贝构造: x=" << x << "\n";
    }
};
​
C c1(5);
C c2 = c1;   // 调用拷贝构造
```

### 4. **移动构造函数（C++11）**

- 作用：用右值（临时对象）初始化，避免不必要的拷贝。

```cpp
class D {
public:
    int* data;
    D(int val) { data = new int(val); }
    // 移动构造
    D(D&& other) noexcept : data(other.data) {
        other.data = nullptr;  // 转移所有权
        std::cout << "移动构造\n";
    }
    ~D() { delete data; }
};
​
D d1(10);
D d2 = std::move(d1);  // 调用移动构造
```

### 5. **赋值运算符（=）**

- **拷贝赋值运算符**

```
class E {
public:
    int x;
    E(int v=0) : x(v) {}
    E& operator=(const E& other) {
        x = other.x;
        std::cout << "拷贝赋值\n";
        return *this;
    }
};
E e1(5), e2(10);
e2 = e1;   // 拷贝赋值
```

## 🔹 1. 左值 (Lvalue)

- **有名字、有地址、可取地址的对象**
- 一般存活在作用域里的一段时间
- 可以出现在赋值号的左边

👉 例子：

```
int a = 10;   // a 是左值（有名字、可取地址）
a = 20;       // 左值可以出现在等号左边
```

------

## 🔹 2. 右值 (Rvalue)

- **没有名字的临时对象**
- 一般是表达式结果、字面量、临时返回值
- 不能取地址，生命周期很短（语句结束销毁）

👉 例子：

```
int b = 5 + 3;   // 5+3 是右值（临时结果）
int c = 42;      // 42 是右值（字面量）
```

------

## 🔹 3. 左值引用 (Lvalue Reference)

- `T&` 只能绑定到左值。
- 不能绑定右值（除非加 `const`）。

👉 例子：

```
int x = 100;
int& ref = x;          // ok，ref 引用左值
const int& cref = 42;  // ok，const 左值引用可以绑定右值
```

------

## 🔹 4. 右值引用 (Rvalue Reference)

- `T&&` 只能绑定到右值。
- 这是 C++11 新引入的，用来支持 **移动语义** 和 **完美转发**。

👉 例子：

```
int&& rref = 42;   // ok，绑定右值
int&& rref2 = x;   // error，不能绑定左值
int&& rref3 = std::move(x);  // ok，move(x) 把左值 x 转成右值
```



## 🔹 多态的分类

1. **静态多态（编译时多态）**
   - **函数重载**：同一个函数名，根据参数个数、类型不同，调用不同版本。
   - **运算符重载**：如 `+`、`<<` 等运算符可以定义为类的行为。
     👉 靠 **编译器** 在编译阶段决定调用哪个函数。
2. **动态多态（运行时多态）**
   - 通过 **虚函数**（`virtual`）机制实现。
   - 父类指针/引用指向子类对象，运行时根据对象的实际类型调用相应函数。
     👉 靠 **虚函数表（vtable）** 和 **动态绑定** 在运行时决定调用哪个函数。

### 虚函数表 wait

虚函数表（**vtable**）是 C++ 实现 **运行时多态** 的核心机制。你前面问到“动态多态 / 引用 / 虚函数表”，正好可以系统讲下：

------

## 1. 为什么需要虚函数表

在 C++ 里，普通函数调用在 **编译期就能确定**（静态绑定）。
 但虚函数需要 **运行时决定到底调用哪一个版本**（动态绑定），这就需要额外的机制来支撑——虚函数表。

------

## 2. 虚函数表的基本结构

编译器在 **含有虚函数的类** 里，会做两件事：

1. **生成一张虚函数表（vtable）**
   - 表里按顺序存储函数指针，每个指针指向类对应的虚函数实现。
   - 如果子类重写了父类的虚函数，表里就会被子类的函数指针覆盖。
2. **在对象里加一个隐藏指针（vptr）**
   - 每个对象都持有一个指向 vtable 的指针。
   - 这样，通过对象的 vptr → vtable → 找到正确的函数实现。

------

## 3. 调用流程

以简单的例子说明：

```
#include <iostream>
using namespace std;

class Base {
public:
    virtual void foo() { cout << "Base::foo\n"; }
    virtual void bar() { cout << "Base::bar\n"; }
};

class Derived : public Base {
public:
    void foo() override { cout << "Derived::foo\n"; }
    void bar() override { cout << "Derived::bar\n"; }
};

int main() {
    Base* p = new Derived();
    p->foo(); // 调用 Derived::foo
    p->bar(); // 调用 Derived::bar
}
```

流程是：

1. `p` 指向 `Derived` 对象。
2. 对象里有一个 `vptr` → 指向 `Derived` 的 vtable。
3. 调用 `p->foo()` 时，通过 `vptr` 找到 vtable 中 `foo` 的函数指针。
4. 跳转到 `Derived::foo` 执行。

------

## 4. 内存示意

假设 `Base` 有两个虚函数，`Derived` 重写了它们：

```
对象内存：
+-----------------+
| data members... |
| vptr ---------->+------> vtable (Derived)
+-----------------+       +------------------+
                         | &Derived::foo    |
                         | &Derived::bar    |
                         +------------------+
```

如果是 `Base` 对象，`vptr` 会指向 `Base` 的 vtable。

------

## 5. 特点

- 只有含虚函数的类才有 vtable。
- `final` 或非虚函数不会出现在 vtable。
- 析构函数如果是虚函数，vtable 里也会有一条析构的入口（保证 delete 基类指针时正确析构子类）。

#### 静态多态

##### 函数重载

```
#include <iostream>
using namespace std;

class Math {
public:
    int add(int a, int b) {
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
};

int main() {
    Math m;
    cout << m.add(1, 2) << endl;       // 调用 int 版本 → 输出 3
    cout << m.add(1.5, 2.5) << endl;   // 调用 double 版本 → 输出 4
}
```

#### 模板（泛型）

```
#include <iostream>
using namespace std;

template<typename T>
T add(T a, T b) {
    return a + b;
}

int main() {
    cout << add(3, 4) << endl;       // 实例化为 int 版本 → 7
    cout << add(2.5, 3.5) << endl;   // 实例化为 double 版本 → 6.0
}
```

#### 动态多态

## 2. 动态多态的语法

核心要素：

1. 基类里声明 **虚函数**（`virtual`）
2. 子类里 **重写（override）** 该虚函数
3. 通过 **基类指针或引用** 调用函数，运行时才决定执行哪一个版本

示例：

```
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { // 虚函数
        cout << "Animal sound" << endl;
    }
    virtual ~Animal() {} // 基类析构函数一般也写成 virtual
};

class Dog : public Animal {
public:
    void speak() override { // 重写
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* a1 = new Dog();
    Animal* a2 = new Cat();

    a1->speak(); // 输出: Woof!
    a2->speak(); // 输出: Meow!

    delete a1;
    delete a2;
    return 0;
}
```

## 示例：用引用实现动态多态

```
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { // 虚函数
        cout << "Animal sound" << endl;
    }
    virtual ~Animal() {}
};

class Dog : public Animal {
public:
    void speak() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "Meow!" << endl;
    }
};

void makeSpeak(Animal& a) { // 注意这里是引用
    a.speak(); // 动态绑定，根据实际对象类型调用
}

int main() {
    Dog d;
    Cat c;

    makeSpeak(d); // 输出: Woof!
    makeSpeak(c); // 输出: Meow!

    return 0;
}
```

## `noexcept` 是什么？

`noexcept` 是 C++11 引入的**异常说明符**（exception specifier），
 用来告诉编译器 **某个函数不会抛出异常**。

## 静态成员（`static` member）

在类中使用 `static` 修饰的成员变量或成员函数，称为**静态成员**。
 它们**属于类本身**，而不是某个具体对象。

## 🧩 一、静态成员变量（`static` 成员变量）

### ✅ 1. 定义方式

静态成员变量属于 **类本身**，而不是属于类的某个对象。

```
class MyClass {
public:
    static int count;  // 声明静态成员变量
};
```

### ✅ 2. 定义与初始化（类外）

必须 **在类外定义一次**，否则会出现链接错误。

```
int MyClass::count = 0;  // 定义并初始化
```

> ⚠️ 注意：
>
> - 静态成员变量 **在所有对象间共享**，只有一份存储空间。
> - 静态成员变量可以在对象创建前访问。

------

### ✅ 3. 使用方式

- 通过 **类名** 或 **对象** 都可以访问。

```
MyClass::count = 5;  // 推荐方式
MyClass obj1;
obj1.count = 10;     // 语法允许，但效果一样
```

### ✅ 4. 生命周期

静态成员变量的生命周期与程序一致（和全局变量类似）。

## ⚙️ 二、静态成员函数（`static` 成员函数）

### ✅ 1. 定义方式

静态成员函数同样属于类本身，而不是某个对象：

```
class MyClass {
public:
    static void showCount();
};

void MyClass::showCount() {
    std::cout << "count = " << count << std::endl;
}
```

> 静态函数不需要对象实例即可调用。

------

### ✅ 2. 调用方式

```
MyClass::showCount();  // 推荐
MyClass obj;
obj.showCount();       // 语法允许
```

------

### ✅ 3. 限制

静态成员函数 **不能访问非静态成员变量或函数**，因为它没有 `this` 指针。

✅ 可以访问：

- 静态成员变量；
- 其他静态成员函数。

❌ 不能访问：

- 非静态成员变量；
- 非静态成员函数。
