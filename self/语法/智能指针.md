智能指针

## 1️⃣ 什么是智能指针

- **智能指针**是 C++ 标准库提供的模板类，用于自动管理动态分配的对象。
- 主要目标：
  1. 自动释放内存（RAII，资源获取即初始化）
  2. 避免手动 `delete` 出错
  3. 提供安全的所有权管理

C++11 主要有三种智能指针：

| 类型            | 作用       | 所有权         | 适用场景                              |
| --------------- | ---------- | -------------- | ------------------------------------- |
| `unique_ptr<T>` | 独占所有权 | 独占           | 不需要共享资源的对象，替代裸指针 `T*` |
| `shared_ptr<T>` | 共享所有权 | 引用计数       | 多个对象需要共享同一块资源            |
| `weak_ptr<T>`   | 弱引用     | 不增加引用计数 | 避免 `shared_ptr` 循环引用            |

------

## 2️⃣ `unique_ptr`（独占指针）

- 独占资源，不能复制，只能移动（`std::move`）
- 内存释放由析构函数自动调用 `delete`

```
#include <memory>
#include <iostream>
using namespace std;

int main() {
    unique_ptr<int> up(new int(42));  // C++14 可用 make_unique
    cout << *up << endl;

    // up2 = up; // ❌ 错误，不能复制
    unique_ptr<int> up2 = std::move(up); // ✅ 转移所有权
}
```

- **推荐用法**：

  ```
  auto p = make_unique<int>(100); // 安全、简洁
  ```

------

## 3️⃣ `shared_ptr`（共享指针）

- 允许多个指针共享同一对象
- 内部维护引用计数，最后一个引用销毁对象
- 可以和 `weak_ptr` 配合，避免循环引用

```
#include <memory>
#include <iostream>
using namespace std;

int main() {
    shared_ptr<int> sp1 = make_shared<int>(42);
    shared_ptr<int> sp2 = sp1; // ✅ 引用计数 +1

    cout << *sp1 << " " << *sp2 << endl;
    cout << sp1.use_count() << endl; // 输出 2
}
```

------

## 4️⃣ `weak_ptr`（弱引用）

- 不增加引用计数
- 用于观察对象或打破循环引用
- 需要通过 `lock()` 获得 `shared_ptr` 才能使用

```
shared_ptr<int> sp = make_shared<int>(42);
weak_ptr<int> wp = sp;

if (auto tmp = wp.lock()) {
    cout << *tmp << endl; // 可以安全使用
}
```

### 🧩 一、什么是循环引用（circular reference）

当两个（或多个）对象通过 `std::shared_ptr` **相互持有**时，
 它们的引用计数永远不会降为 0，从而导致**内存无法释放（内存泄漏）**。

📘 例子：

```
struct B; // 前向声明

struct A {
    std::shared_ptr<B> b_ptr;
    ~A() { std::cout << "A destroyed\n"; }
};

struct B {
    std::shared_ptr<A> a_ptr;
    ~B() { std::cout << "B destroyed\n"; }
};

int main() {
    auto a = std::make_shared<A>();
    auto b = std::make_shared<B>();
    a->b_ptr = b;
    b->a_ptr = a;
}
```

输出：

```
（无输出）
```

👉 因为 `a` 和 `b` 相互引用，`use_count` 永远是 2，无法释放。

------

### 🔁 二、为什么会出现循环引用

- `std::shared_ptr` 会**增加引用计数**。
- 当 `A` 拥有 `B` 的 `shared_ptr`，而 `B` 又拥有 `A` 的 `shared_ptr` 时：
  - 两者都认为对方“还活着”，引用计数永远大于 0。
  - 析构函数永远不会被调用，造成**资源泄漏**。

------

### 🧯 三、解决方案：使用 `std::weak_ptr`

`std::weak_ptr` 是一个**不增加引用计数**的弱引用。
 它不会影响对象的生命周期，只是“观察”一个 `shared_ptr` 所指的对象。

修改例子：

```
struct B; 

struct A {
    std::shared_ptr<B> b_ptr;
    ~A() { std::cout << "A destroyed\n"; }
};

struct B {
    std::weak_ptr<A> a_ptr;  // 👈 改成 weak_ptr
    ~B() { std::cout << "B destroyed\n"; }
};

int main() {
    auto a = std::make_shared<A>();
    auto b = std::make_shared<B>();
    a->b_ptr = b;
    b->a_ptr = a;
}
```

输出：

```
A destroyed
B destroyed
```

✅ 这样就能正常析构，不会内存泄漏。

------

### 🧠 四、`weak_ptr` 的几个要点

1. **不会增加引用计数**（`use_count` 不变）。

2. 使用前要通过 `.lock()` 获得一个 `shared_ptr`：

   ```
   if (auto sp = weak_a.lock()) {
       // 对象仍存在，可以安全使用
   }
   ```

3. 常用于：

   - 解决循环引用。
   - 缓存、观察者模式等场景（不控制对象生命周期）。

------

### ⚙️ 五、总结要点表

| 类型              | 是否增加引用计数 | 是否影响生命周期 | 用途                 |
| ----------------- | ---------------- | ---------------- | -------------------- |
| `std::shared_ptr` | ✅ 是             | ✅ 是             | 共享所有权           |
| `std::weak_ptr`   | ❌ 否             | ❌ 否             | 避免循环引用、弱引用 |
| `std::unique_ptr` | ❌ 否（唯一）     | ✅ 是             | 独占所有权           |
