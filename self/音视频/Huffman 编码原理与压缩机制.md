# Huffman 编码原理与压缩机制

## 1️⃣ 核心思想

Huffman 编码是一种 **可变长度前缀编码（Variable-Length Prefix Coding）**，压缩原理是：

- 出现频率高的符号 → 用短比特表示
- 出现频率低的符号 → 用长比特表示
- 通过这种方式，使整体比特数最少 → 达到压缩效果

------

## 2️⃣ 压缩步骤概览

1. **统计符号频率**
   - 输入数据序列 → 统计每个符号出现次数
2. **构造 Huffman 树**
   - 将每个符号作为叶子节点，权重 = 符号频率
   - 每次选择两个频率最小的节点合并 → 形成父节点
   - 重复直到只剩一个根节点
3. **生成编码表**
   - 从根节点到叶子节点的路径 → 符号的比特编码
   - 路径左/右可用 0/1 表示
   - 保证**前缀唯一**（任意码不可能是另一个码的前缀）
4. **编码数据**
   - 用生成的 Huffman 表，将原始符号替换为比特序列
   - 拼接所有比特 → 压缩后的比特流
5. **解码数据**
   - 从比特流逐步解析，根据 Huffman 树或表还原符号

------

## 3️⃣ 举例说明

假设要编码的符号序列：

```
A A A B B C
```

### 步骤 1：统计频率

- A = 3
- B = 2
- C = 1

### 步骤 2：构造 Huffman 树

```
        (*6)
       /    \
   (*3)      A
   /   \
  C     B
```

### 步骤 3：生成编码表

| 符号 | Huffman 编码 |
| ---- | ------------ |
| A    | 1            |
| B    | 01           |
| C    | 00           |

### 步骤 4：编码数据

原序列：`A A A B B C`

编码后比特流：`1 1 1 01 01 00`

- 固定长度编码（比如每个符号 2 bit）需要 6 × 2 = 12 bit
- Huffman 编码只用了 9 bit → 节省 25% 比特
