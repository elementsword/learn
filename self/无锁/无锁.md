```cpp
#include <iostream>
#include <atomic>
#include <thread>
#include <vector>

template <typename T>
class LockFreeQueue {
private:
    struct Node {
        T data;
        std::atomic<Node*> next;
        Node(T data) : data(data), next(nullptr) {}
    };

    std::atomic<Node*> head;
    std::atomic<Node*> tail;

public:
    LockFreeQueue() {
        Node* dummy = new Node(T());  // 创建一个虚拟节点作为头和尾
        head.store(dummy);
        tail.store(dummy);
    }

    ~LockFreeQueue() {
        // 清理内存
        while (head.load() != nullptr) {
            Node* tmp = head.load();
            head.store(tmp->next);
            delete tmp;
        }
    }

    void enqueue(T data) {
        Node* newNode = new Node(data);
        Node* oldTail = tail.load();
        Node* nullNode = nullptr;

        // 使用原子操作确保队尾更新是无锁的
        while (!oldTail->next.compare_exchange_weak(nullNode, newNode)) {
            oldTail = tail.load();
        }

        // 更新尾部指针
        tail.compare_exchange_weak(oldTail, newNode);
    }

    bool dequeue(T& result) {
        Node* oldHead = head.load();
        Node* newHead = oldHead->next;

        // 队列为空时返回 false
        if (newHead == nullptr) {
            return false;
        }

        // 从队列头部取出数据
        result = newHead->data;

        // 更新头指针
        head.compare_exchange_weak(oldHead, newHead);
        delete oldHead;  // 删除已经出队的节点

        return true;
    }
};

// 测试无锁队列
void producer(LockFreeQueue<int>& queue) {
    for (int i = 0; i < 10; ++i) {
        queue.enqueue(i);
        std::cout << "Produced: " << i << std::endl;
    }
}

void consumer(LockFreeQueue<int>& queue) {
    int value;
    for (int i = 0; i < 10; ++i) {
        while (!queue.dequeue(value)) {
            std::this_thread::yield();  // 如果队列为空，等待下一轮
        }
        std::cout << "Consumed: " << value << std::endl;
    }
}

int main() {
    LockFreeQueue<int> queue;

    // 创建生产者和消费者线程
    std::thread producerThread(producer, std::ref(queue));
    std::thread consumerThread(consumer, std::ref(queue));

    producerThread.join();
    consumerThread.join();

    return 0;
}
```

### bool compare_exchange_weak(T& expected, T desired,

​                           std::memory_order success,
​                           std::memory_order failure) noexcept;

## 核心功能

`compare_exchange_weak` 做的事情本质上是 **原子 CAS（Compare-And-Swap）操作**：

1. 读取当前原子变量值 `current`
2. 比较 `current == expected`：
   - 如果相等：
     - 将原子变量值更新为 `desired`
     - 返回 `true`
   - 如果不等：
     - 把 `expected` 更新为 `current`
     - 返回 `false`

> weak 的特点是，它可能**虚假失败（spuriously fail）**，即使 `current == expected`，也可能返回 `false`。

### bool compare_exchange_strong(T& expected, T desired,

​                             std::memory_order success,
​                             std::memory_order failure) noexcept;

## 核心功能

`compare_exchange_strong` 做的事情本质上是 **CAS（Compare-And-Swap）**：

1. 读取当前原子变量值 `current`
2. 比较 `current == expected`：
   - 如果相等 → 更新为 `desired`，返回 true
   - 如果不等 → 不更新 atomic_var，将 `expected = current`，返回 false

> 与 `weak` 最大的不同是 **不会出现虚假失败**。只要值相等，就一定成功。

## 2️⃣ 并发情况分类

假设有 **线程 A 和线程 B 同时 enqueue**。

### 情况 1：两个线程几乎同时操作相同 tail 节点

```
Initial:
tail -> Node0 -> nullptr
```

- A 和 B 都读取 `oldTail = Node0`。
- A 先执行 `compare_exchange_weak` 成功，把 `Node0->next` 指向 `NodeA`。
- B 执行 `compare_exchange_weak` 失败（因为 `Node0->next != nullptr`）。
- B 会重新加载 `tail`，发现 `tail` 可能还在 `Node0` 或已经被 A 更新成 `NodeA`。
- B 再尝试 `compare_exchange_weak`，现在 `oldTail->next` 是 `NodeA`，B 会继续向后找到真正的队尾。
- 最终 B 成功把 `NodeA->next` 指向 `NodeB`。

✅ 保证链表顺序正确，尾部指针在最后通过 CAS 更新。

------

### 情况 2：tail 指针已被其他线程更新

```
Initial:
tail -> Node0 -> nullptr

线程 A enqueue NodeA
线程 B enqueue NodeB
```

- A 成功把 `Node0->next = NodeA`，再成功更新 `tail = NodeA`。
- B 在 while 循环中：
  1. `oldTail->next` 失败（因为 `oldTail = Node0->next != nullptr`）
  2. 重新加载 `tail = NodeA`
  3. 再尝试 `NodeA->next.compare_exchange_weak(nullptr, NodeB)` 成功
  4. 更新 `tail = NodeB`

✅ 结果：链表顺序仍然是 Node0 -> NodeA -> NodeB

------

### 情况 3：多个线程几乎同时竞争，形成“跳跃尾”

假设有 3 个线程 T1/T2/T3 同时 enqueue：

```
tail -> Node0 -> nullptr
```

- T1 成功把 Node0->next = Node1
- T2 compare_exchange_weak 失败，重新加载 tail = Node1，Node1->next = nullptr
- T3 compare_exchange_weak 失败，重新加载 tail = Node1
- T2 成功把 Node1->next = Node2
- T3 再次循环，发现 Node2->next = nullptr，最终把 Node2->next = Node3
- tail 最终依次被 T1/T2/T3 更新到 Node3

✅ 保证每个节点都被正确链接，没有丢失或覆盖。

```
// mpmc_michael_scott.cpp
// g++ -std=c++17 mpmc_michael_scott.cpp -O2 -pthread -o mpmc_test

#include <atomic>
#include <iostream>
#include <thread>
#include <vector>
#include <cassert>

template<typename T>
class MPMCQueue {
private:
    struct Node {
        std::atomic<Node*> next;
        T data;
        // two ctors: dummy node and value node
        Node() : next(nullptr), data() {}
        Node(const T& v) : next(nullptr), data(v) {}
    };

    std::atomic<Node*> head; // points to dummy or oldest node
    std::atomic<Node*> tail; // hint to tail

public:
    MPMCQueue() {
        Node* dummy = new Node();
        head.store(dummy, std::memory_order_relaxed);
        tail.store(dummy, std::memory_order_relaxed);
    }

    ~MPMCQueue() {
        // Note: If other threads still run, this is unsafe.
        T tmp;
        while (dequeue(tmp));
        Node* p = head.load(std::memory_order_relaxed);
        delete p;
    }

    // enqueue: multiple producers safe
    void enqueue(const T& value) {
        Node* newNode = new Node(value);
        while (true) {
            Node* last = tail.load(std::memory_order_acquire);
            Node* next = last->next.load(std::memory_order_acquire);
            // check consistency
            if (last == tail.load(std::memory_order_acquire)) {
                if (next == nullptr) {
                    // last is really the last node, try link newNode
                    if (last->next.compare_exchange_weak(next, newNode,
                                                         std::memory_order_release,
                                                         std::memory_order_relaxed)) {
                        // try to swing tail to newNode (optimization; may fail)
                        tail.compare_exchange_weak(last, newNode,
                                                   std::memory_order_release,
                                                   std::memory_order_relaxed);
                        return;
                    }
                    // failed to link (virtual failure or raced), retry
                } else {
                    // tail is behind, help advance it
                    tail.compare_exchange_weak(last, next,
                                               std::memory_order_release,
                                               std::memory_order_relaxed);
                }
            }
        }
    }

    // dequeue: multiple consumers safe
    // returns false if queue empty
    bool dequeue(T& result) {
        while (true) {
            Node* first = head.load(std::memory_order_acquire);
            Node* last  = tail.load(std::memory_order_acquire);
            Node* next  = first->next.load(std::memory_order_acquire);

            if (first == head.load(std::memory_order_acquire)) {
                if (next == nullptr) {
                    // empty
                    return false;
                }
                if (first == last) {
                    // tail falling behind, try to advance it
                    tail.compare_exchange_weak(last, next,
                                               std::memory_order_release,
                                               std::memory_order_relaxed);
                    continue;
                }
                // read value before CAS to avoid ABA on next
                result = next->data;
                // try to move head forward
                if (head.compare_exchange_weak(first, next,
                                               std::memory_order_release,
                                               std::memory_order_relaxed)) {
                    // successful, safe to reclaim old head
                    delete first;
                    return true;
                }
                // else failed, retry
            }
        }
    }

    // utility: check empty (not atomic with other ops)
    bool empty() const {
        Node* first = head.load(std::memory_order_acquire);
        Node* next = first->next.load(std::memory_order_acquire);
        return (next == nullptr);
    }
};

// Simple test: multiple producers and multiple consumers
int main() {
    MPMCQueue<int> q;

    const int producers = 4;
    const int consumers = 4;
    const int per_prod = 1000;
    const int total = producers * per_prod;

    std::atomic<int> produced{0};
    std::atomic<int> consumed{0};

    // producers
    std::vector<std::thread> pts;
    for (int i = 0; i < producers; ++i) {
        pts.emplace_back([&q,&produced,i,per_prod](){
            for (int j = 0; j < per_prod; ++j) {
                int v = i * per_prod + j;
                q.enqueue(v);
                produced.fetch_add(1, std::memory_order_relaxed);
            }
        });
    }

    // consumers
    std::vector<std::thread> cts;
    for (int i = 0; i < consumers; ++i) {
        cts.emplace_back([&q,&consumed,total](){
            int v;
            while (consumed.load(std::memory_order_relaxed) < total) {
                if (q.dequeue(v)) {
                    consumed.fetch_add(1, std::memory_order_relaxed);
                    // optionally process v
                } else {
                    std::this_thread::yield();
                }
            }
        });
    }

    for (auto &t : pts) t.join();
    for (auto &t : cts) t.join();

    std::cout << "produced=" << produced.load() << " consumed=" << consumed.load() << "\n";
    assert(produced.load() == consumed.load());
    std::cout << "test passed\n";
    return 0;
}
```

# enqueue

## 📜 入队操作流程

### 1️⃣ 创建新节点

- 分配一个新的节点对象 `newNode`
- 填入待入队的值
- `newNode->next` 初始化为 `nullptr`

> 因为它将成为新的“最后一个节点”。

------

### 2️⃣ 读取当前尾节点

- 从 `tail` 原子变量中读取当前的尾指针，称为 `last`。
- 然后读取 `last->next`，即尾节点的下一个指针。

此时要判断：

- `next == nullptr`：说明 `last` 是真正的尾节点；
- `next != nullptr`：说明有其他线程正在插入节点，`tail` 可能滞后。

------

### 3️⃣ 检查一致性

- 再次检查 `tail` 是否仍然等于刚才读取的 `last`；
- 若不一致，说明其他线程更新了 `tail`，当前线程重试。

> 这是为了避免读到“中间状态”的结构。

------

### 4️⃣ 如果 `next == nullptr`

**说明当前尾节点确实是最后一个节点：**

- 尝试把新节点挂在 `last->next` 上；
- 使用原子比较交换（CAS）操作实现：
  - 若成功，表示当前线程成功将 `newNode` 链接到了队列末尾；
  - 若失败（被别人抢先插入或虚假失败），则重试。

成功后：

- 可顺便尝试更新 `tail = newNode`，这只是性能优化，不更新也不影响正确性。

------

### 5️⃣ 如果 `next != nullptr`

**说明尾指针滞后：**

- 可能别的线程已经插入了节点，但还没更新 `tail`；
- 当前线程“帮忙”推进 `tail = next`；
- 然后重新循环尝试入队。

------

### 6️⃣ 重试循环

整个操作都在一个无限循环中：

- 任何一步竞争失败都会重新读取最新的 `tail` 和 `next`；
- 当成功链接新节点后才返回。

#  Dequeue（出队）操作流程

------

## 1️⃣ 读取当前头、尾、下一节点

- 从 `head` 原子变量中读取当前的头指针 `first`。
- 读取 `tail`（尾指针）和 `first->next`（头节点的下一个）。

> 这里 `first` 是“哨兵节点”（dummy node），真正的数据在 `first->next` 里。

------

## 2️⃣ 一致性检查

- 再次确认 `head` 没被别的线程改动（即 `first` 仍是当前 `head`）；
- 如果不一致，则说明有并发修改，重新开始循环。

> 这一检查防止在出队过程中读到结构中间态。

------

## 3️⃣ 判断是否为空队列

- 若 `next == nullptr`：
  - 表示队列中没有数据节点（`first` 是最后一个节点）；
  - 出队失败，返回 `false`。

------

## 4️⃣ 处理尾指针滞后

- 若 `first == last` 且 `next != nullptr`：

  - 表示 `tail` 滞后了（有新节点已插入但 `tail` 未更新）；

  - 当前线程尝试帮助推进：

    ```
    tail = next
    ```

  - 然后重新开始循环，继续尝试出队。

> 这种“帮助推进”是无锁队列常见的自修复机制，保证结构一致性。

------

## 5️⃣ 读取要出队的数据

- 当 `next != nullptr` 且结构一致时，
  - 从 `next->data` 中读取即将出队的值，保存到 `result`。

> 提前读取是为了避免 ABA 问题：
>  如果等到 CAS 成功后再读数据，节点可能已被其他线程复用或释放。

------

## 6️⃣ 尝试推进 `head`

- 使用 CAS 将 `head` 从 `first` 更新为 `next`：
  - 若成功：
    - 表示当前线程成功出队；
    - 安全地释放旧的 `first` 节点（哨兵节点）。
  - 若失败：
    - 说明其他线程同时出队，`head` 已被更新；
    - 放弃并重新循环。

------

## 7️⃣ 重试与结束

- 若出队成功，返回 `true`；
- 若任何一步失败（CAS 失败或结构变化），重新读取 `head`、`tail`、`next` 并重试。
