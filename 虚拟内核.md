安装依赖

配置和编译 kernel 源码

make menuconfig

在 menuconfig 的菜单中，我们需要启用以下选项以便调试内核：


Kernel hacking ---> 
  [*] Kernel debugging
  Compile-time checks and compiler options --->
    [*] Compile the kernel with debug info
    [*]  Provide GDB scripts for kernel debugging
关闭随机化内核地址(KASLR)选项，因为它会使得调试过程更加复杂：

Processor type and features ---->
   [] Randomize the address of the kernel image (KASLR)

 

然后使用 make 命令编译内核。




make -j 20

cp linux-5.15/arch/x86/boot/bzImage .

编译kernel主要是为了获取到bzImage和vmlinux文件

使用qemu运行内核

qemu直接运行如下：
qemu-system-x86_64 -kernel ./arch/x86_64/boot/bzImage -hda ./busybox-1.32.0/rootfs.img -append "root=/dev/sda console=ttyS0" -nographic

Ctrl+A 松开后按C退出qemu。




qemu调试运行如下：
一个终端：
qemu-system-x86_64 -kernel ./arch/x86_64/boot/bzImage -hda ./busybox-1.32.0/rootfs.img -append "root=/dev/sda console=ttyS0" -s -S -smp 1 -nographic

另外一个终端，启动gdb远程调试：
gdb ./vmlinux （编译完后，kernel源码目录下将会出现该文件）
(gdb) target remote localhost:1234

qemu调试加共享磁盘如下：
qemu-system-x86_64 -kernel ./arch/x86_64/boot/bzImage -hda ./busybox-1.32.0/rootfs.img -append "root=/dev/sda console=ttyS0" -s -smp 1 -nographic -hdb ./ext4.img

kernel后面是之前编译出来的bzImage -hda后面是之前制作的根文件系统，-hdb后面的是直接创建的共享磁盘

然后在原本的机器上挂载共享磁盘
sudo mount -t ext4 -o loop ext4.img ./share

在qemu中也需要挂载共享磁盘（即/dev/sdb设备）
mount /dev/sdb /mnt
